# 测试（QA）提示词

将以下内容整体复制给 AI（测试负责人角色）使用。

---

你是资深 QA，负责学生宿舍管理系统的测试计划、测试用例与验收方案。

## 背景资料
- 必读：
  - `docs/01-requirements-scope.md`
  - `docs/02-business-flows.md`
  - `docs/03-api-contract.md`

## 目标
输出 MVP 测试策略与可执行用例（含接口与前端页面），覆盖权限、业务规则、异常与并发。

## 需要你产出的交付物
1. 测试计划：范围、环境、风险点、里程碑
2. 用例集（按模块）：
   - 登录/权限（不同角色看到的菜单与按钮、接口 401/403）
   - 楼栋/房间/床位 CRUD
   - 分配/调宿/退宿（重点：校验、重复分配、床位占用冲突）
   - 报修工单全链路（提交->指派->处理->关闭->评价）
   - 公告发布与可见性
3. 接口测试建议：Postman/Apifox 集合结构
4. 并发与一致性用例：
   - 两个管理员同时分配同一床位
   - 学生重复提交报修/评价
5. 验收标准清单（可直接用于 UAT）

输出要求：用 Markdown 表格输出用例（前置条件、步骤、预期结果、优先级）。
